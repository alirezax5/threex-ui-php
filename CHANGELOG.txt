# Changelog

All notable changes to the 3X-UI PHP Client are documented in this file.

---

## [1.0.0] ‚Äî 2026-07-07

### Added
#### Core
- Initial release of `alirezax5/threex-ui-php`
- `ThreeXUI\ThreeXUI` main facade with fluent interface
- `ThreeXUI\Config` ‚Äî immutable-style configuration builder
- `ThreeXUI\HttpClient` ‚Äî native cURL HTTP client with JSON handling
- `ThreeXUI\Authentication` ‚Äî dual auth: Bearer token + session login
- Cookie-based session management with auto CSRF token fetch

#### Endpoints (9 groups, ~96 methods)
- `Endpoints\Inbounds` ‚Äî full CRUD, traffic reset, fallback management, bulk import
- `Endpoints\Clients` ‚Äî CRUD, bulk ops (create/delete/attach/detach/adjust/reset), traffic, IPs, online status
- `Endpoints\ClientGroups` ‚Äî group CRUD, bulk add/remove members
- `Endpoints\Server` ‚Äî status monitoring, Xray service control, key generation (UUID, X25519, ML-DSA, ML-KEM, ECH), logs, DB backup/import, geo file updates
- `Endpoints\Nodes` ‚Äî multi-node cluster CRUD, connectivity test/probe, metric history
- `Endpoints\Settings` ‚Äî panel settings, user credentials update, API token management
- `Endpoints\XrayConfig` ‚Äî config template, outbound traffic stats, Warp/NordVPN integration, outbound testing
- `Endpoints\CustomGeo` ‚Äî custom geoip/geosite source management
- `Endpoints\Subscriptions` ‚Äî Base64/JSON/Clash subscriptions, Telegram backup

#### Helpers
- `Helpers\Validator` ‚Äî email, UUID, port, protocol, positive int, non-empty array validation
- `Helpers\Formatter` ‚Äî bytes‚áÑhuman, GB‚áÑbytes, timestamp, expiry, truncate, sanitize
- `Helpers\ArrayHelper` ‚Äî dot notation get/set/has, remove empty, only/except, to query
- `Helpers\RetryHandler` ‚Äî exponential backoff with configurable jitter
- `Helpers\functions.php` ‚Äî 9 global helper functions (threexui_client, bytes_to_human, etc.)

#### Exceptions
- `Exceptions\ThreeXUIException` ‚Äî base exception with context array
- `Exceptions\AuthenticationException` ‚Äî auth failures (401)
- `Exceptions\ApiException` ‚Äî API error responses (4xx/5xx) with responseData
- `Exceptions\ConnectionException` ‚Äî network/cURL errors
- `Exceptions\ValidationException` ‚Äî input validation failures

#### Retry Logic
- Retry on: ConnectionException, HTTP 408/429/5xx
- No retry on: AuthenticationException, 4xx client errors
- Configurable: maxAttempts, baseDelayMs, multiplier, maxDelayMs, jitter
- `$panel->withRetry()` fluent method on facade
- `HttpClient::setRetryHandler()` for programmatic control

#### Contracts
- `Contracts\HttpClientInterface` ‚Äî HTTP abstraction
- `Contracts\EndpointInterface` ‚Äî endpoint marker

#### Testing
- PHPUnit 10 test suite: 144 tests, 226 assertions
- Test files: Config, Exceptions, Validator, Formatter, ArrayHelper, RetryHandler, Functions
- Endpoint tests: Inbounds, Clients, ClientGroups, Server, Nodes, Settings, XrayConfig, CustomGeo
- Mocked HTTP layer for endpoint testing

#### Documentation

- `DOCUMENTATION.fa.md` ‚Äî full Persian API reference

#### Examples (7 files)
- `examples/basic-usage.php` ‚Äî authentication and basic API calls
- `examples/inbounds.php` ‚Äî complete inbound management
- `examples/clients.php` ‚Äî client CRUD + bulk + groups
- `examples/server.php` ‚Äî monitoring, service control, keygen, logs, backups
- `examples/nodes.php` ‚Äî multi-node cluster management
- `examples/settings.php` ‚Äî panel settings + API token management
- `examples/xray-config.php` ‚Äî Xray core config, Warp/NordVPN
- `examples/advanced.php` ‚Äî error handling, 2FA, retry, maintenance scripts
- `examples/helpers.php` ‚Äî all helper functions demonstrated
