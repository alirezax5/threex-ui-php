# 3X-UI PHP Client — Full LLM Context

## Project Overview

**alirezax5/threex-ui-php** is a comprehensive, modular PHP 8.1+ client library for the 3X-UI Panel API. The 3X-UI Panel is a web-based management interface for Xray-core, providing GUI control over V2Ray/VLESS/VMess/Trojan/Shadowsocks proxy configurations. This library wraps all ~96 REST API endpoints across 9 functional groups with type-safe, validated, and well-structured PHP code.

### Target Audience
- Developers automating 3X-UI panel management
- VPN service providers building custom client portals
- System administrators needing scriptable panel control
- AI/LLM agents generating or manipulating panel configurations

## Architecture Deep Dive

### Core Classes

#### `ThreeXUI\ThreeXUI` (Main Facade)
The primary entry point. Constructor accepts a panel URL string. Provides:
- `withApiToken(string $token): self` — Bearer token auth
- `login(string $user, string $pass): array` — Session login
- `logout(): array` — Session logout
- `isTwoFactorEnabled(): bool` — Check 2FA
- `setTimeout(int $seconds): self` — Request timeout
- `withoutSslVerification(): self` — Skip SSL (dev only)
- Accessor methods for each endpoint group: `inbounds()`, `clients()`, `clientGroups()`, `server()`, `nodes()`, `settings()`, `xrayConfig()`, `customGeo()`, `subscriptions()`
- `getHttpClient(): HttpClient` — Direct HTTP access
- `getConfig(): Config` — Configuration access

#### `ThreeXUI\Config`
Immutable-style configuration with fluent interface:
- `__construct(string $baseUrl)` — Sets panel URL
- `useApiToken(string $token): self` — Switch to token mode
- `useCredentials(string $user, string $pass): self` — Switch to session mode
- `setTimeout(int $seconds): self`
- `disableSslVerification(): self`
- Getters: `getBaseUrl()`, `getApiToken()`, `getUsername()`, `getPassword()`, `getTimeout()`, `isSslVerified()`

#### `ThreeXUI\HttpClient`
Native cURL HTTP client implementing `HttpClientInterface`:
- `get(string $path, array $query): array`
- `post(string $path, array $data): array`
- `postMultipart(string $path, array $data, array $files): array`
- `put(string $path, array $data): array`
- `delete(string $path): array`
- Automatic: JSON encoding/decoding, header management, cookie extraction
- Adds `Authorization: Bearer` header when API token is set
- Adds `Cookie:` header for session-based auth
- Response handling: parses 3X-UI JSON envelope, throws exceptions on errors

#### `ThreeXUI\Authentication`
Handles the authentication lifecycle:
- `login(): array` — POST /login, captures session cookie
- `logout(): array` — POST /logout, clears session
- `getCsrfToken(): ?string` — Returns stored CSRF token
- `isTwoFactorEnabled(): bool` — POST /getTwoFactorEnable
- Auto-fetches CSRF token after successful login

### Exception Hierarchy

- `ThreeXUIException` extends `\RuntimeException`
  - `AuthenticationException` — 401, invalid creds/token
  - `ApiException` — Any API error response, stores `getResponseData()`
  - `ConnectionException` — cURL/network failures
  - `ValidationException` — Input validation failures

All exceptions carry optional `$context` array via `getContext()`.

## Usage Patterns for LLM Agents

### Creating an inbound with Reality
```php
$panel = new \ThreeXUI\ThreeXUI('https://panel:54321');
$panel->withApiToken('token');

$keys = $panel->server()->getNewX25519Cert();

$result = $panel->inbounds()->add([
    'remark'          => 'VLESS-Reality',
    'listen'          => '',
    'port'            => 443,
    'protocol'        => 'vless',
    'enable'          => true,
    'total'           => 0,
    'expiryTime'      => 0,
    'settings'        => json_encode([
        'clients'    => [],
        'decryption' => 'none',
        'fallbacks'  => []
    ]),
    'streamSettings'  => json_encode([
        'network'     => 'tcp',
        'security'    => 'reality',
        'realitySettings' => [
            'show'         => false,
            'xver'         => 0,
            'dest'         => 'www.microsoft.com:443',
            'serverNames'  => ['www.microsoft.com'],
            'privateKey'   => $keys['obj']['privateKey'] ?? '',
            'shortIds'     => [''],
        ]
    ]),
    'sniffing'        => json_encode([
        'enabled'      => true,
        'destOverride' => ['http', 'tls']
    ]),
]);
```

### Batch client management
```php
// Get all depleted clients and remove them
$panel->clients()->delDepleted();

// Add days to premium users' expiry
$premiumEmails = $panel->clientGroups()->emails('premium');
$panel->clients()->bulkAdjust($premiumEmails['obj'] ?? [], addDays: 30);

// Create bulk clients from CSV data
$entries = array_map(function($row) {
    return [
        'client'     => ['email' => $row[0], 'totalGB' => (int) $row[1]],
        'inboundIds' => [1],
    ];
}, $csvRows);
$panel->clients()->bulkCreate($entries);
```

### Server monitoring
```php
$status = $panel->server()->status();
// $status['obj']['cpu'] — CPU usage %
// $status['obj']['mem']['current'] — Current memory
// $status['obj']['disk']['current'] — Disk usage
// $status['obj']['xray']['state'] — Xray running state
// $status['obj']['netIO']['up'] / ['down'] — Network I/O

// Get logs
$logs = $panel->server()->logs(50, level: 'error');
$xrayLogs = $panel->server()->xrayLogs(100);
```
